RINGKASAN PROGRES TERBARU
================================================================

Per 22 April 2026, modul KPI personal dan 360 sudah mengalami beberapa
perubahan penting dibanding handover lama:

1. Workflow KPI manager-level sudah ditambahkan.
   - Head Store dinilai oleh District Manager + Support Supervisor,
     diverifikasi Area Manager.
   - District Manager dinilai oleh Area Manager + Support Supervisor,
     diverifikasi Ops Manager.
   - Jika District Manager tidak punya Area Manager (mis. area tertentu),
     penilaian dilakukan Ops Manager + Support Supervisor,
     diverifikasi Ops Manager.

2. Login store team sudah kembali memakai model hybrid.
   - Staff/store team dapat login dengan email + password bersama
     dari VITE_STAFF_PASS.
   - Head Store dan manager tetap login memakai email + password akun.

3. KPI personal dan 360 sekarang punya fallback frontend.
   - Jika seed item di database belum lengkap, frontend tetap bisa
     menampilkan item default melalui src/lib/kpiDefaults.js.

4. Root cause kosongnya KPI/360 di production sudah ditemukan.
   - BUKAN karena tanggal 5.
   - BUKAN karena item 360 kosong.
   - Penyebab utama: RLS policy profiles terlalu sempit, sehingga
     store roles tidak bisa membaca daftar profile rekan/tim untuk KPI.
   - Fix database sudah disiapkan di migration 017.

5. Struktur repo sudah dirapikan.
   - File referensi dipindah ke reference/
   - File handover dipindah ke docs/handover/
   - Script generator dipindah ke scripts/
   - Peta struktur repo ditulis di docs/PROJECT_MAP.md

================================================================
ROOT CAUSE KPI/360 KOSONG DI PRODUCTION
================================================================

Investigasi tanggal 22 April 2026 menunjukkan:

- kpi_360_items aktif ada:
  - store = 5
  - manager = 5
- banyak head_store punya visible_store_peers > 0
- banyak store profile valid dengan branch_id aktif
- ada juga 42 akun role 'staff'
- ada 13 akun store-level dengan branch_id NULL

Kesimpulan:

1. Item penilaian bukan masalah utama.
2. Periode/tanggal 5 bukan penyebab kosongnya halaman.
3. Penyebab utama adalah policy SELECT pada tabel profiles terlalu sempit.

Fix yang disiapkan:
- supabase/migrations/017_profiles_kpi_visibility.sql

Isi fix 017:
- Store roles bisa baca profile store roles di branch yang sama
- Head Store bisa baca District Manager / Area Manager yang relevan
  untuk 360 grup manager

SQL diagnosis yang dipakai:
- supabase/diagnose_kpi_visibility.sql
