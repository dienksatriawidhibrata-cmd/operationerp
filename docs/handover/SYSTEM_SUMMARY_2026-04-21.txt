DATABASE SCHEMA (Supabase PostgreSQL)
================================================================

Semua tabel menggunakan Row Level Security (RLS).
Semua tabel ada kolom id UUID DEFAULT gen_random_uuid().
Helper functions di DB: my_role() dan my_branch() tersedia di RLS.

── TABEL UTAMA ─────────────────────────────────────────────────

profiles
  id UUID (FK ke auth.users)
  full_name TEXT
  role TEXT (salah satu dari 21 role di atas)
  branch_id UUID → branches
  is_active BOOLEAN
  managed_districts TEXT[] (khusus district_manager)
  managed_areas TEXT[]     (khusus area_manager)
  branch JSONB (di-join saat query, berisi nama branch)

branches
  id UUID
  name TEXT ('Bagi Kopi Cipete', dll)
  district TEXT
  area TEXT
  is_active BOOLEAN

announcements
  id, title, body, published_at, is_active

── OPERASIONAL TOKO ────────────────────────────────────────────

daily_checklists
  id, branch_id, tanggal DATE, shift TEXT ('pagi'|'middle'|'malam')
  submitted_by UUID, submitted_at TIMESTAMPTZ
  is_late BOOLEAN (via TRIGGER, bukan generated column)
  Deadline: pagi=08:00WIB, middle=14:00WIB, malam=03:00WIB+1
  UNIQUE (branch_id, tanggal, shift)

checklist_items
  id, item_name, section TEXT, shift TEXT
  ('pagi'|'malam'|'both'|'middle'|'pagi_middle')
  is_active BOOLEAN, sort_order INT

daily_checklist_responses
  id, checklist_id UUID → daily_checklists
  item_id UUID → checklist_items
  is_done BOOLEAN

daily_preparation
  id, branch_id, tanggal DATE, shift TEXT ('pagi'|'middle'|'malam')
  submitted_by UUID, notes TEXT
  UNIQUE (branch_id, tanggal, shift)

daily_preparation_items
  id, preparation_id UUID → daily_preparation
  item_key TEXT, section TEXT ('bar'|'kitchen'), qty NUMERIC

preparation_config
  item_key TEXT, item_name TEXT, section TEXT, unit TEXT, sort_order INT

daily_reports
  id, branch_id, tanggal DATE, submitted_by UUID
  submitted_at TIMESTAMPTZ, is_late BOOLEAN
  total_pax INT, omzet NUMERIC, catatan TEXT

operational_expenses
  id, branch_id, tanggal DATE, kategori TEXT
  deskripsi TEXT, total NUMERIC, submitted_by UUID

── KPI TOKO ────────────────────────────────────────────────────

kpi_monthly_reports
  id, branch_id, period_month TEXT ('YYYY-MM')
  [banyak kolom metrik toko]

── KPI PERSONAL + 360° (migration 013) ─────────────────────────

kpi_personal_items                        ← CONFIG TABLE
  id, role TEXT, item_key TEXT, item_name TEXT
  target TEXT, contribution INT (bobot %, total per role = 100)
  cara_penilaian TEXT, source_type TEXT
    CHECK IN ('auto_checklist','auto_preparation','auto_360','manual')
  score_1..score_5 TEXT (deskripsi per skor)
  is_active BOOLEAN, sort_order INT
  UNIQUE (role, item_key)

  !! PENTING: kolom bobot bernama 'contribution', BUKAN 'weight_pct' !!
  !! cara_penilaian adalah deskripsi/description item, BUKAN 'description' !!

kpi_360_items                             ← CONFIG TABLE
  id, group_type TEXT CHECK IN ('store','manager')
  item_key TEXT, item_name TEXT, description TEXT
  score_1..score_5 TEXT
  is_active BOOLEAN, sort_order INT
  UNIQUE (group_type, item_key)

kpi_personal_scores                       ← DATA TABLE
  id, staff_id UUID → profiles
  branch_id UUID → branches
  period_month TEXT ('YYYY-MM')
  item_key TEXT, auto_value NUMERIC(6,2)
  score INT CHECK 1-5
  notes TEXT, scored_by UUID → profiles
  verified_by UUID → profiles, verified_at TIMESTAMPTZ
  UNIQUE (staff_id, period_month, item_key)

  RLS:
  SELECT: staff sendiri | head_store/asst (branch_id sama) | DM/AM/ops/support_spv/trainer
  INSERT: head_store saja (branch_id = my_branch())
  UPDATE: head_store (jika belum verified_at) | DM/AM/ops

kpi_360_submissions                       ← DATA TABLE
  id, evaluator_id UUID → profiles
  evaluatee_id UUID → profiles
  branch_id UUID → branches
  group_type TEXT CHECK IN ('store','manager')
  period_month TEXT, catatan TEXT (max 1000 char)
  CONSTRAINT no_self_eval CHECK (evaluator_id != evaluatee_id)
  UNIQUE (evaluator_id, evaluatee_id, period_month)

kpi_360_scores                            ← DATA TABLE
  id, submission_id UUID → kpi_360_submissions ON DELETE CASCADE
  item_key TEXT, score INT CHECK 1-5
  UNIQUE (submission_id, item_key)

CATATAN TEKNIS KRITIS UNTUK CODEX
================================================================

1. NAMA KOLOM WAJIB BENAR:
   kpi_personal_items.contribution  (BUKAN weight_pct)
   kpi_personal_items.cara_penilaian (BUKAN description)
   kpi_360_items.description         (ini BENAR, pakai description)

2. TIMEZONE:
   Semua perhitungan tanggal pakai WIB (UTC+7)
   todayWIB(), currentPeriodWIB() sudah handle ini
   DB simpan timestamp sebagai UTC (Supabase default)

3. is_late di daily_checklists:
   WAJIB via TRIGGER, bukan generated column
   (PostgreSQL tidak bisa pakai AT TIME ZONE di generated column)

4. Upsert pattern:
   Semua submit form pakai .upsert(..., { onConflict: '...' })
   Bukan insert → jika sudah ada akan update (edit friendly)

5. 360° anonimitas:
   evaluator_id TERSIMPAN di DB di semua kondisi
   Frontend HANYA tampilkan nama evaluator jika profile.role === 'ops_manager'
   Jangan expose evaluator_id ke role lain

6. Score calculation untuk KPI Personal:
   Auto checklist = pctToScore(jumlah ceklis bulan itu / (daysInMonth × 3) × 100)
   Auto prep      = pctToScore(jumlah prep bulan itu / (daysInMonth × 3) × 100)
   Auto 360       = avg360ToScore(rata-rata semua kpi_360_scores untuk staff itu bulan itu)
   Ini dihitung di frontend (KPIPersonalInputPage) saat form dibuka,
   BUKAN disimpan sebagai auto_value di DB (kolom ada tapi tidak dipakai saat ini)

7. Kolega filtering 360°:
   Store group  → supabase.from('profiles').eq('branch_id', ...).in('role', store_roles)
   Manager group (head_store punya) → query branches.district/area dulu,
     lalu .contains('managed_districts', [district])
   DM group → query branches by managed_districts, lalu head_store .in('branch_id', ids)
   AM group → query branches by managed_areas, lalu head_store .in('branch_id', ids)

8. period_month format: 'YYYY-MM' string, bukan DATE
   Semua tabel KPI pakai TEXT untuk period_month

9. Build clean:
   npm run build harus selesai tanpa error sebelum push ke main

10. Cloudflare env vars tidak bisa diubah via code:
    Hanya bisa ubah di Cloudflare dashboard
