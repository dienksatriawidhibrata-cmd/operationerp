# 📊 Pemetaan Komprehensif Web Apps Bagikopi Ops

## 1️⃣ Sistem Autentikasi

| Komponen | Detail |
|----------|--------|
| **Provider** | Supabase Auth |
| **Login Methods** | Email + Password (admin/manager) atau Email + Staff Password (staff toko) |
| **Session** | Cache di localStorage dengan timeout verification 8 detik |

---

## 2️⃣ 14 Role & Hierarki Akses

### **Store Roles** (karyawan toko)
| Role | Akses |
|------|-------|
| `staff`, `barista`, `kitchen`, `waitress` | Dashboard, ceklis, SOP, KPI personal |
| `asst_head_store` | + Laporan, QC |
| `head_store` | + Pengajuan opex, people management, input KPI |

### **Manager Roles** (manajemen regional)
| Role | Scope |
|------|-------|
| `district_manager` | Districts di `managed_districts` |
| `area_manager` | Areas di `managed_areas` |
| `ops_manager` | Semua branches |

### **Support Roles** (pusat)
| Role | Fungsi |
|------|--------|
| `support_spv`, `support_admin` | Semua branches, full access |

### **Functional Roles**
| Role | Deskripsi |
|------|-----------|
| `auditor` | Audit setoran |
| `finance_supervisor` | Keuangan & opex |
| `trainer` | Training staff |
| `hr_staff`, `hr_spv`, `hr_legal`, `hr_administrator` | Recruitment & HR |
| `warehouse_admin`, `warehouse_spv`, `picking_spv`, `qc_spv`, `distribution_spv`, `purchasing_admin`, `sc_supervisor` | Supply Chain |

## 4️⃣ Runtutan Flow Mengakses

### 🔄 Flow Universal (Semua Role)

```
/login
    ↓
Cek session & profile (loading state)
    ↓
Root redirect berdasarkan role:
├── STORE_ROLES → /staff
├── DM / Area Manager → /dm
├── OPS / Support → /ops
├── TRAINER → /trainer
├── FINANCE → /finance
├── HR → /hr
└── SC roles → /sc/[picking|qc|distribution]
```

## 5️⃣ Mekanisme Proteksi Route

Setiap halaman menggunakan `<RequireAuth>` dengan urutan cek:

1. **Loading** → "Sesi masih aktif..."
2. **Authentication** → Redirect ke `/login` jika tidak login
3. **Profile** → "Menghubungkan data profil..."
4. **Active Status** → Tampilkan "Akun Dinonaktifkan" jika `is_active === false`
5. **Authorization** → Redirect ke `/` jika role tidak sesuai
6. **Render** → Lazy load dengan Suspense

### Data Scoping per Role:

| Role | Scope Data |
|------|------------|
| Store | Hanya branch_id mereka |
| District Manager | Hanya managed_districts |
| Area Manager | Hanya managed_areas |
| Ops/Support | Semua branches (no filter) |
