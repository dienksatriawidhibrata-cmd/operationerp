12. RECRUITMENT MODULE
═══════════════════════════════════════════════════════════════
Status: LIVE di lokal (testing confirmed), belum deploy ke production.
Routes: /hr, /hr/batch, /hr/batch/:id, /hr/candidates/:id,
        /hr/candidates/:id/ojt, /hr/store, /hr/kontrak

ROLES BARU:
  hr_staff       → buat batch, review form, forward ke hr_spv
  hr_spv         → approval final sebelum kontrak
  hr_legal       → submit kontrak → trigger auto-create akun
  hr_administrator → semua aksi HR (super-HR, hanya di modul recruitment)
  Login: individual password via tab "Head Store & Manager"

STATE MACHINE (14 stage):
  batch_oje_issued      → HS/DM upload nilai batch
  batch_oje_uploaded    → hr_staff seleksi siapa lanjut
  batch_oje_reviewed    → hr_staff terbitkan OJE in Store
  oje_instore_issued    → HS upload form kelulusan OJE
  oje_instore_submitted → hr_staff review form
  review_hrstaff        → approve ke hr_spv / flag revisi ke HS
  revision_hs           → HS edit hanya field yang diflag
  pending_hrspv         → hr_spv approve / kembalikan ke hr_staff
  revision_hrstaff      → hr_staff putuskan langkah berikutnya
  kontrak_pending       → hr_legal submit kontrak
    → AUTO: buat auth user + profile (role & branch dari data kandidat)
    → AUTO: kandidat masuk stage ojt_instore, status active
  ojt_instore           → HS + Trainer isi OJT Checklist (14 item)
  assessment            → Trainer assess (pass/fail, max 2x)
  training              → Trainer complete
  on_duty               → aktif bekerja, status on_duty

OJT CHECKLIST (src/pages/hr/OjtChecklist.jsx):
  - 14 item, 3 kolom: Head Store | PIC Training | Staff
  - Tick per item sesuai kolom role yang login
  - Final sign-off per pihak (tombol aktif setelah semua item di-tick)
  - is_complete = true jika ketiga pihak sudah sign-off

TABEL SUPABASE:
  candidates          — pipeline record, source of truth stage
  stage_history       — audit trail append-only (tidak boleh delete/update)
  stage_forms         — form data versioned (is_current pattern)
  ojt_checklists      — checklist OJT per kandidat (JSONB per item)
  oje_batches         — batch header (existing, reused)
  oje_batch_items     — peserta batch (existing, reused)
  oje_evaluations     — evaluasi individual (existing, reused)

EDGE FUNCTION:
  supabase/functions/recruitment-onboard/index.ts
  → Dipanggil frontend saat hr_legal submit kontrak
  → Butuh Supabase Secret: STAFF_SHARED_PASSWORD
  → Belum di-deploy (lokal only)
